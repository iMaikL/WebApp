Best practices voor het vak FrontEnd 2

1. Gebruik comments om je code duidelijker te maken voor jezelf en voor een ander
2. Maak namen in je code doormiddel van camelCasing (als je meerdere woorden achter elkaar plakt is het eerste woord geen hoofdletter en de woorden daarna wel)
3. Maak verschillende .js bestanden zodat je overzicht behoud over je code. (vb. hammer.js animatie)
4. Gebruik een namespace, een globale variabele. 
5. Zorg ervoor dat je code niet met elkaar in verwarring kan komen, gebruik daarom een self invoking anonymous function
6. Sla data op in localStorage om de app eerst te laten laden en daarna soepel te laten lopen
7. Plaats scripts onderaan je pagina, zodat de pagina eerst ingeladen wordt en daarna de scripts. 
8. Design je website zodat deze ook goed werkt ZONDER javascript
9. Begrijp de termen (of zet deze in je file) en lees veel over javascript

In deze ReadMe heb ik geen best practices uit de les (zoals het gebruik van transparency en routie) meegenomen. Dit zijn best practices die voor mij als beginner erg belangrijk zijn. 

flow:
1: MOVIEAPP word gemaakt 
2: Data word in localstorage gezet en zorgt dat die bruikbaar is
3: routie ziet wat de persoon gekozen heeft
4: transparency rendert de pagina op met content
5: de change functie in router verandert de route en laat de pagina zien
